# 项目改动说明

## 1. 背景

根据前一轮检查，项目存在两类主要问题：

1. 功能缺失
   - 组织详情接口缺失
   - 组织文档强制归档接口缺失
   - 组织审计导出接口缺失
   - 前端图片上传调用缺失对应 API

2. 前后端协议不一致
   - 邀请接受/拒绝使用了错误参数
   - 协作者管理中误用了权限记录 id 作为用户 id
   - 权限模板、系统配置、统计、组织概览、审计日志等模块存在字段名不一致
   - 多个页面仍按旧的数据结构读取 `owner`、`actor`、`department` 等对象

本次修改目标是优先打通“系统管理员、组织管理员、文档创作者、文档协作者”相关核心闭环，使主要功能可以正常使用。

---

## 2. 本次完成的主要修改

### 2.1 协作邀请与协作者管理

已修复：

- 邀请接受/拒绝改为使用邀请 `token`
- 邀请列表接口补充分页能力
- 邀请列表返回补充 `inviter_name`
- 协作者管理改为使用 `user_id` 更新/删除协作者
- 协作者角色补充 `commenter`
- 协作者列表返回补充 `is_owner`
- 分享链接生成统一使用 `share_token`

影响：

- 文档创作者可以正常邀请协作者
- 文档协作者可以正常接受/拒绝邀请
- 协作者角色修改、移除、查看流程恢复正常

涉及文件：

- `server/routes/collaboration.js`
- `server/middleware/permissions.js`
- `client/src/components/DocPermModal.vue`
- `client/src/views/collab/Invitations.vue`
- `client/src/views/collab/SharedDocs.vue`
- `client/src/views/Dashboard.vue`
- `client/src/api/collaboration.ts`

### 2.2 系统管理员模块修复

已修复：

- 补充 `GET /api/admin/orgs/:id` 组织详情接口

- 系统配置接口兼容前端直接提交对象和 `{ configs }` 包裹结构

影响：

- 系统概览页统计可以正常显示
- 组织管理页详情弹窗可正常打开
- 权限模板管理基本可用
- 系统配置可正常读取和保存
- 注册开关与配置链路对齐

涉及文件：

### 2.3 组织管理员模块修复

影响：

- 组织概览页统计恢复正常

### 2.4 文档查看、分享、评论、导出链路修复

已修复：

## 4. 目前仍保留的限制

以下内容本次未继续深入扩展：

1. “四个基本角色”仍主要表现为：
   - 系统管理员
   - 组织管理员
   - 普通用户
   - 文档级权限角色（creator / editor / commenter / viewer）

   也就是说，“文档创作者、文档协作者”目前主要是文档权限身份，而不是独立系统角色。

2. PDF 导出仍未实现
   - 当前仅支持 HTML/JSON

3. 一些页面仍属于“基础可用”，但还可以继续增强
   - 组织协作规则配置仍未做成独立模块
   - 组织审计目前更偏操作审计，不是严格意义上的版本实体审计
   - 仪表盘中的部分业务指标仍可进一步细化

---

## 5. 总结

本次修改重点不是新增全新模块，而是把原本“有页面、有接口但对不上”的部分打通，使项目的核心业务链路恢复可用，尤其是：

- 系统管理员管理链路
- 组织管理员统计与审计链路
- 文档创作者邀请协作链路
- 文档协作者接受邀请、查看共享文档链路
- 文档查看、评论、分享、上传图片等常用功能

如果后续还需要继续完善，可以优先从以下方向继续：

1. 强化四类角色为真正的独立角色体系
2. 完整实现 PDF/Word 导出
3. 将组织协作规则配置做成独立可配置模块
4. 对组织审计补充更细粒度的版本审计模型
